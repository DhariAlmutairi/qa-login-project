## 1. Introduction
This document describes the test plan for the Login functionality of a web application.
The purpose of this testing is to ensure that the login feature works correctly,
securely, and provides a good user experience.

## 2. Scope

### In Scope
- Login with valid email and password
- Login with invalid credentials
- Validation messages
- UI behavior of login page
- Error handling
- Session handling (basic)

### Out of Scope
- Performance testing
- Load testing
- Security penetration testing

## 3. Test Objectives
- Verify that users can log in using valid credentials
- Ensure error messages are displayed correctly for invalid inputs
- Validate mandatory field behavior
- Ensure system prevents unauthorized access

## 4. Test Approach
- Manual Testing
- Black Box Testing
- Functional Testing
- Regression Testing

## 5. Test Environment
- Browser: Google Chrome (latest version)
- Operating System: macOS
- Test URL: https://the-internet.herokuapp.com/login

## 6. Test Deliverables
- Test Plan
- Test Scenarios
- Test Cases
- Bug Reports
- Regression Checklist

## 7. Entry Criteria
- Application is accessible
- Test environment is ready
- Requirements are stable

## 8. Exit Criteria
- All test cases are executed
- All critical and high severity bugs are fixed or documented
- Test summary is completed

## 9. Risks & Mitigation
| Risk | Mitigation |
|-----|------------|
| Requirements not clear | Clarify before execution |
| Environment issues | Use stable test environment |

## 10. Roles & Responsibilities
- QA Engineer: Test planning, execution, and reporting
- Developer: Bug fixing
- Product Owner: Requirement clarification
